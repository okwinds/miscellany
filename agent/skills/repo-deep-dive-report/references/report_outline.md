# 报告大纲模板（可直接复制到 `docs/repo_review.md`）

> 目标：把“仓库走读”写成可复用的交付件：能定位代码、能跑起来、能二开、能评估质量。

## 01. 项目概览
- 仓库定位（从 README/代码形态推断）
- 关键目录/边界（apps/services/packages, core, docs, examples, tests, infra）
- 快速指标（规模、语言、关键依赖）

## 02. 架构地图（分层 + 核心对象 + 数据流）
- 模块清单（按目录分组）
- 核心抽象与关系（谁依赖谁、通过接口还是事件连接）
- Mermaid：模块依赖图（graph TD）

## 03. 入口与执行流程（从锚点开始追）
- 锚点说明（入口/函数/路由/CLI 子命令）
- 初始化与装配（settings/config/DI/plugin registry）
- 关键链路步骤（逐步列出方法/文件路径）
- Mermaid：时序图（sequenceDiagram，可选）

## 04. 核心模块深挖（3–6 个高杠杆模块）
对每个模块固定结构：
- 概念与职责
- 代码定位（文件/类/关键函数）
- 核心数据结构
- 关键流程/算法（错误处理/重试/并发/缓存）
- 扩展点（如何新增/替换/注入）
- 最小示例（伪代码/真实示例）

## 05. 上手实操（本地跑起来）
- 最小依赖与关键依赖（分类）
- 最小配置（env/config 文件/密钥/base_url）
- 跑通一个 example/最小路径（命令 + 预期输出）
- 常见坑与排查（网络/权限/流式/超时/版本）

## 06. 二次开发指南（可操作清单）
- 新增一个自定义模块/插件/工具/节点：改哪些文件？遵循什么模式？
- 扩展校验规则/解析器：插入点在哪里？如何不破坏兼容？
- 如何添加最小测试/示例来验证扩展

## 07. 仓库文档总结
- 组织方式建议
- 核心 API 使用规范
- 鼓励/禁止的模式与原因

## 08. 评分（100 分制，多维度）
- 总分 + 分项分数 + 证据点 + 改进建议（按影响/成本排序）

## 09. 附录：关键文件/符号速查
- 入口点列表
- 核心抽象清单
- 配置键/环境变量速查（如有）
